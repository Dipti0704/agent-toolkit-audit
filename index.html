The agent

What actually did the research, and where it needed a human.
01

Pass 1 — wide net

Per app: an OpenAI tool-use loop with Composio's no-auth COMPOSIO_SEARCH toolkit actually searches and reads the app's real docs, then a structured-extraction call turns the notes into a validated record.

→
02

Pass 2 — blind re-check

Every app is independently re-researched from scratch, in a fresh session with no exposure to pass 1's answers. Field-by-field agreement is the first accuracy signal.

→
03

Reconcile

Agreement keeps the value with high confidence. Disagreement is flagged, demoted in confidence, and surfaced — not silently averaged away.

→
04

Human spot-check

A fixed random 20-app sample is hand-checked against real docs. This is the only step that produces a real ground-truth accuracy number.

→
05

Patterns

Pure aggregation over the final dataset — no LLM call. Auth distribution, self-serve % by category, easy wins, most common blocker.

Where a human was needed

The LLM does the searching, fetching, and structured extraction end to end. A human is required for (a) the ground-truth check against real docs on the 20-app sample — two independent LLM passes agreeing is not proof of correctness, both can be confidently wrong the same way — and (b) judgment calls on genuinely ambiguous gating language that neither pass could resolve. Everything else in this pipeline ran unattended.

OpenAI · gpt-5.4-nano Composio · COMPOSIO_SEARCH (no-auth toolkit) Structured outputs (Pydantic schema) Python · concurrent research pipeline
